Factory Inventory Management System

System architecture, tech stack, and data flow

A demo full-stack application for managing factory inventory across three global warehouses (San Francisco, London, Tokyo). It gives operations teams a single dashboard to monitor stock levels, track orders through their fulfilment lifecycle, review procurement spending, and surface demand forecasts and backlog shortfalls. All data is mock and held in memory — the system is designed as a realistic prototype for demonstrating GitHub and AI-assisted development workflows, not as a production deployment.

Filter Data Flow
1
User selects a filter in FilterBar.vue
Four dropdowns: Time Period, Warehouse, Category, Order Status. Values bound via v-model to useFilters() composable refs.
2
Composable state updates, views react
Each view watches the filter refs via watch(). Any change automatically triggers a reload function.
3
api.js builds the HTTP request
getCurrentFilters() returns {warehouse, category, status, month}. Axios appends non-"all" values as query params.
4
FastAPI receives and filters
apply_filters() chains warehouse → category → status checks. filter_by_month() handles YYYY-MM and Q1-2025 formats. All comparisons are case-insensitive.
5
Pydantic validates, JSON returned
Filtered list validated against response model. Frontend receives pre-filtered JSON and re-renders the view.
